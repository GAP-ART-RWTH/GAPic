  
  [1X3 [33X[0;0YAnimating surfaces with Java Script[133X[101X
  
  [33X[0;0YThis  chapter  deals with animating simplicial surfaces via Javascript using
  three.js, see [7Xhttps://threejs.org/[107X. Currently we are using r148.[133X
  
  
  [1X3.1 [33X[0;0YIntroduction and Quick Start[133X[101X
  
  [33X[0;0YThis section contains a minimal example by animating an octahedron and shows
  the  general animation workflow. To construct an octahedron, we only need to
  specify the 3D-coordinates of the vertices.[133X
  
  [4X[32X  Example  [32X[104X
    [4X[25Xgap>[125X [27Xoct := Octahedron();;[127X[104X
    [4X[25Xgap>[125X [27XverticesPositions := [[127X[104X
    [4X[25X>[125X [27X[ 0, 0, Sqrt(2.) ],[127X[104X
    [4X[25X>[125X [27X[ 1, 1, 0 ],[127X[104X
    [4X[25X>[125X [27X[ 1, -1, 0 ],[127X[104X
    [4X[25X>[125X [27X[ -1, -1, 0 ],[127X[104X
    [4X[25X>[125X [27X[ -1, 1, 0 ],[127X[104X
    [4X[25X>[125X [27X[ 0, 0, -Sqrt(2.) ] ];;[127X[104X
    [4X[25Xgap>[125X [27XprintRecord := SetVertexCoordinates3D(oct, verticesPositions, rec());;[127X[104X
    [4X[25Xgap>[125X [27XDrawComplexToJavascript(oct, "octahedron.html", printRecord);;[127X[104X
  [4X[32X[104X
  
  [33X[0;0YNow, your working directory should contain a file "octahedron.html". Here is
  an image of the animation:[133X
  [33X[0;0Y[Image omitted in text manual. Check doc/images/Octahedron_animated.png][133X
  [33X[0;0YThe philosophy of the code presented in this chapter is as follows: The main
  method [9XDrawComplexToJavascript[109X (Section [14X3.3[114X) animates the surfaces using the
  configuration  saved  in  the so called [3XprintRecord[103X, e.g. the 3D-coordinates
  (lists  with  three float entries) of the vertices. It is necessary that the
  printRecord  contains the 3D-coordinates of each vertex of the surface which
  should be animated. Based on the 3D-coordinates of the vertices, the surface
  is  placed  in  the  animation.  Therefore,  a typical workflow to animate a
  surfaces looks like this:[133X
  
  [31X1[131X   [33X[0;6YConstruct the surface, e.g. the octahedron above.[133X
  
  [31X2[131X   [33X[0;6YConfigure  your  animation  using  the methods in the Sections [14X3.4[114X and
        [14X3.5[114X.[133X
  
  [31X3[131X   [33X[0;6YWrite   your   animation   to   a   file   calling   the  main  method
        [9XDrawComplexToJavascript[109X (see Section [14X3.3[114X).[133X
  
  [33X[0;0YThe  the method [9XDrawComplexToJavascript[109X([14X3.3-3[114X) creates and html file in your
  current  location.  The  file  can be opened with most browsers and then the
  surface  is  shown as a 3D render with some manipulation options. Because we
  load  the  three.js  dependecies  (and  three.js itself) from a CDN (content
  delivery network) it requires an active internet connection.[133X

  (GUI).   Here   is   an  example:  [Image  omitted  in  text  manual.  Check
  doc/images/GUI.png][133X
  [33X[0;0YHere you can change the following:[133X
  
  [31X1[131X   [33X[0;6YThe speed of rotation in the X, Y and Z direction[133X

  [31X3[131X   [33X[0;6YShow the edges according to visibility and colour[133X
  
  [31X4[131X   [33X[0;6YThe  visibility  of the active edges and their respective labels. More
        in [14X3.4[114X[133X
  
  [31X5[131X   [33X[0;6YThe size the vertices are show with[133X
  
  [31X6[131X   [33X[0;6YSwitch to normalsMaterial. More in [14X3.8[114X[133X
  
  [31X7[131X   [33X[0;6YOptional:    Parameters    set    for   the   coordinates.   More   in

  [33X[0;0YThis GUI is implemented using the dat.GUI package from three.js.[133X
  
  
  [1X3.2-1 [33X[0;0YIntersection Planes[133X[101X
  
  [33X[0;0YIn  the GUI there is the option for so called intersection planes. These are
  three  axis  aligned  planes, that intersect the complete animation and only
  show one of the sides.[133X
  [33X[0;0YThis  allows  to look into the animation and can be very helpful for example
  if there are selfintersections or a complex gets very big.[133X
  [33X[0;0YTo  activate  any plane, toggle the respective checkbox in the GUI. Directly
  underneath, there is a slider to adjust where the plane should intersect the
  animation.[133X
  [33X[0;0YIf      there      are      not     parameterized     vertex     coordinates
  ([14X???Section_ParameterizedCoordinates???[114X)   the   sliders  are  automatically
  adjusted so you can intersect the whole complex. If you want more precision,
  either  make the GUI larger by dragging in the left of the GUI or change the
  slider by typing a number in the field next to it.[133X
  [33X[0;0YIf  there  are coordinates which are parameterized or you want to change the
  range of the planes for another reason, it is possible to set them manually:[133X
  
  [1X3.2-2 SetPlaneRange[101X
  
  [33X[1;0Y[29X[2XSetPlaneRange[102X( [3Xsurface[103X, [3Xranges[103X, [3XprintRecord[103X ) [32X operation[133X
  [6XReturns:[106X  [33X[0;10Ythe updated print record[133X
  
  [33X[0;0YChange  the  range of the intersection planes by giving a list of the [3Xranges[103X
  in the following format: [3X[[minX, maxX],[minY, maxY],[minZ, maxZ]][103X[133X
  
  [1X3.2-3 GetPlaneRange[101X
  
  [33X[1;0Y[29X[2XGetPlaneRange[102X( [3Xsurface[103X, [3XprintRecord[103X ) [32X operation[133X
  [6XReturns:[106X  [33X[0;10Ya list of lists[133X
  
  [33X[0;0YReturns  the  plane  ranges  set  in  [3XSetPlaneRange[103X  in  the  format [3X[[minX,
  maxX],[minY,  maxY],[minZ,  maxZ]][103X  or  [3Xfail[103X  if  they  are  not  set  (i.e.
  automatic).[133X
  
  
  [1X3.3 [33X[0;0YCoordinates and Core Functionality[133X[101X
  
  [33X[0;0YThis  section  describes  the  animation  method  and  how  to define vertex
  coordinates. Before the animation (see [9XDrawComplexToJavascript[109X [14X3.3-3[114X) can be
  drawn,  it  is  necessary  to  specify  the 3D-coordinates (which means, the
  [x,y,z] coordinates) of all vertices of the surface which should be animated
  (see [9XSetVertexCoordinates3D[109X [14X3.3-1[114X).[133X
  
  [33X[0;0YAfter  setting  the vertex coordinates, the location of the edges in [22XR^3[122X has
  to  be derived. By default, the method [9XDrawComplexToJavascript[109X([14X3.3-3[114X) always
  calculates  the  edge locations depending on the current vertices positions.
  If    you    want    to    avoid    that,    you    can   use   the   method
  [9XDrawComplexToJavascriptCalculate[109X([14X3.3-3[114X) and set the last parameter to false.
  In this case, only the first call of the method computes the edge locations.
  Using  this method can make the animation inconsistent if the coordinates of
  the vertices have changed but the edge locations have stayed the same.[133X
  
  [33X[0;0YThe  animation  draws  all faces and a wireframe for them individually. This
  wireframe  can  be  enabled and disabled in the user interface. The vertices
  will  be  drawn as small spheres, with the corresponding number is displayed
  above  the  vertex.  These  will  be  only  drawn  if  they  are active, see
  [9XActivateVertices[109X [14X3.4-1[114X[133X
  
  [33X[0;0YNote,  it  is  also  possible to animate simplicial surfaces whose vertices,
  edges  and  faces  are  not  given  by  a dense list [9X[1..n][109X. For example the
  function  [9XSetVertexCoordinates3D[109X  allows  a  list of 3D-coordinates as input
  that is not dense. But this only works in the case that the 3D-coordinate of
  vertex [9Xi[109X is stored in the [9Xi[109X-th position of the given list.[133X
  
  [4X[32X  Example  [32X[104X
    [4X[25Xgap>[125X [27XoneFace:=SimplicialSurfaceByDownwardIncidence([,[3,7],,[7,10],,[3,10]],[127X[104X
    [4X[25X>[125X [27X[,,[2,4,6]]);;[127X[104X
    [4X[25Xgap>[125X [27XVertices(oneFace);[127X[104X
    [4X[28X[ 3, 7, 10 ][128X[104X
    [4X[25Xgap>[125X [27Xcoor:=[];;[127X[104X
    [4X[25Xgap>[125X [27Xcoor[3]:=[1,0,0];;coor[7]:=[0,1,0];;coor[10]:=[0,0,1];;[127X[104X
    [4X[25Xgap>[125X [27XprintRecord:=SetVertexCoordinates3D(oneFace,coor,rec());;[127X[104X
    [4X[25Xgap>[125X [27XDrawComplexToJavascript(oneFace,"OneFace_animating",printRecord);[127X[104X
    [4X[28Xrec( edgeThickness := 0.03, [128X[104X
    [4X[28X vertexCoordinates3D := [ ,, [ 1, 0, 0 ],,,, [ 0, 1, 0 ],,, [ 0, 0, 1 ] ] )[128X[104X
  [4X[32X[104X
  
  [33X[0;0Y[Image omitted in text manual. Check doc/images/OneFace_animated.png][133X
  
  [1X3.3-1 SetVertexCoordinates3D[101X
  
  [33X[1;0Y[29X[2XSetVertexCoordinates3D[102X( [3Xsurface[103X, [3Xcoordinates[103X[, [3XprintRecord[103X] ) [32X operation[133X
  [33X[1;0Y[29X[2XSetVertexCoordinates3DNC[102X( [3Xsurface[103X, [3Xcoordinates[103X[, [3XprintRecord[103X] ) [32X operation[133X
  [6XReturns:[106X  [33X[0;10Ythe updated print record[133X
  
  [33X[0;0YSave  the  given list of 3D-coordinates [3Xcoordinates[103X in the given or an empty
  print  record.  The  list  [3Xcoordinates[103X  has  to have entries that are a list
  [3X[x,y,z][103X  of  three  floats. If the format of the [3Xcoordinates[103X is not correct,
  then an error is shown. The NC-version does not check the coordinate format.[133X
  
  [1X3.3-2 GetVertexCoordinates3D[101X
  
  [33X[1;0Y[29X[2XGetVertexCoordinates3D[102X( [3Xsurface[103X, [3Xvertex[103X, [3XprintRecord[103X ) [32X operation[133X
  [33X[1;0Y[29X[2XGetVertexCoordinates3DNC[102X( [3Xsurface[103X, [3Xvertex[103X, [3XprintRecord[103X ) [32X operation[133X
  [6XReturns:[106X  [33X[0;10Ya list[133X
  
  [33X[0;0YExtract  the  3D-coordinates from the print record of the vertex [3Xvertex[103X from
  [3Xsurface[103X. The 3D-coordinates of vertex [3Xvertex[103X has to have the format [3X[x,y,z][103X.
  If  the  format  of  the [3Xcoordinates[103X is not correct, then an error is shown.
  This  can happen, if the NC version is used to store the 3D-coordinates. The
  NC-version does not check the coordinate format saved in the print record.[133X
  
  [1X3.3-3 DrawComplexToJavascript[101X
  
  [33X[1;0Y[29X[2XDrawComplexToJavascript[102X( [3Xsurface[103X, [3Xfilename[103X, [3XprintRecord[103X ) [32X operation[133X
  [6XReturns:[106X  [33X[0;10Ythe updated print record[133X
  
  [33X[0;0YThese methods animate the [3Xsurface[103X as a html file [3Xfilename[103X in JavaScript into
  the  working  directory.  The  animation can be opened and watched with most
  browsers.  An  introduction to the use of this method with an example can be
  found at the start of section [14X3.1[114X.[133X
  
  [30X    [33X[0;6YIf  the  given [3XfileName[103X does not end in [13X.html[113X the ending [13X.html[113X will be
        added to it.[133X
  
  [30X    [33X[0;6YThe  given  file  will  be  overwritten  without  asking if it already
        exists.  If  you  don't  have  permission  to write in that file, this
        method will throw an error.[133X
  
  [30X    [33X[0;6YThe   particulars   of   the  drawing  are  determined  by  the  given
        [3XprintRecord[103X.[133X
  
  [33X[0;0YTo  use  these  methods  it  is  necessary  to set the 3D-coordinates of the
  vertices of the surface (see [9XSetVertexCoordinates3D[109X).[133X
  
  [33X[0;0YThere  are two parameters to change the output of this method. There are the
  following classes of parameters:[133X
  
  [30X    [33X[0;6Y[13XVisibility[113X ([14X3.4[114X): Change the visibility of vertices, edges and faces.[133X
  
  [30X    [33X[0;6Y[13XColours[113X ([14X3.5[114X): Change the colours of vertices, edges and faces.[133X
  
  [33X[0;0YThere  are also options for inner circles ([14X3.6[114X) and normals of inner circles
  ([14X3.7[114X).[133X
  
  [1X3.3-4 CalculateParametersOfEdges[101X
  
  [33X[1;0Y[29X[2XCalculateParametersOfEdges[102X( [3Xsurface[103X, [3XprintRecord[103X ) [32X operation[133X
  [6XReturns:[106X  [33X[0;10Ythe updated print record[133X
  
  [33X[0;0YThis  method  calculates the parameters of the edges (centre, length, angles
  relative  to x-direction) based on the coordinates of the vertices and saves
  those to the print record.[133X
  
  
  [1X3.4 [33X[0;0YVisibility[133X[101X
  
  [33X[0;0YA  simplicial  surface  is described through its vertices, edges, and faces.
  But  sometimes, not all of them should be visible. Therefore, it is possible
  to make the following visible and invisible:[133X
  
  [31X1[131X   [33X[0;6YVertices (see [14X3.4-1[114X)[133X
  
  [31X2[131X   [33X[0;6YEdges (The default colour is 0x000000, black. See [14X3.5-3[114X)[133X
  
  [31X3[131X   [33X[0;6YFaces (see [14X3.4-3[114X)[133X
  
  [33X[0;0YBy  default,  all vertices, and faces are visible. For the visibility of the
  edges, there is a wireframe option in the GUI of the output file. It is also
  possible  in  the GUI to change the transparency of all faces. Otherwise all
  vertices  which  are  active  will  be  shown.  Both can be changed live and
  updates  the  shown  render  immediately.  The  following  example  shows an
  animation with invisible vertices.[133X
  
  [4X[32X  Example  [32X[104X
    [4X[25Xgap>[125X [27Xoct := Octahedron();;[127X[104X
    [4X[25Xgap>[125X [27XverticesPositions := [[127X[104X
    [4X[25X>[125X [27X[ 0, 0, Sqrt(2.) ],[127X[104X
    [4X[25X>[125X [27X[ 1, 1, 0 ],[127X[104X
    [4X[25X>[125X [27X[ 1, -1, 0 ],[127X[104X
    [4X[25X>[125X [27X[ -1, -1, 0 ],[127X[104X
    [4X[25X>[125X [27X[ -1, 1, 0 ],[127X[104X
    [4X[25X>[125X [27X[ 0, 0, -Sqrt(2.) ] ];;[127X[104X
    [4X[25Xgap>[125X [27XprintRecord := SetVertexCoordinates3D(oct, verticesPositions, rec());;[127X[104X
    [4X[25Xgap>[125X [27XprintRecord := DeactivateVertices(oct, printRecord);;[127X[104X
    [4X[25Xgap>[125X [27XDrawComplexToJavascript(oct, "octahedron.html", printRecord);;[127X[104X
  [4X[32X[104X
  
  [33X[0;0Y[Image         omitted         in         text         manual.         Check
  doc/images/Octahedron_animatedWithoutVertices.png][133X

  [1X3.5 [33X[0;0YColouring[133X[101X
  
  [33X[0;0YIn  this  section,  we describe how to colour the animation. The colours are
  stored   in   hexadecimal  format,  which  starts  with  0x.  The  following
  possibilities are available:[133X
  
  [31X1[131X   [33X[0;6YVertices (The default colour is 0xF58137, an orange hue. See [14X3.5-1[114X)[133X
  
  [31X2[131X   [33X[0;6YFaces (The default colour is 0x049EF4, a light blue hue. See [14X3.5-5[114X)[133X
  
  [33X[0;0YSome colours can also be referred to by strings, namely:[133X
